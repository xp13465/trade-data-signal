## 一、参考案例：上证指数 sh（已完成打样）

### 1.1 回测参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 总资金 | 100,000 元 | 固定 |
| 固定操作金额 | 10,000 元 | 路径 A/C 每次买入金额 |
| 最多同时持仓 | 10 笔 | 10w/1w |
| 成交价 | 信号当日收盘价 | 无滑点假设 |
| 信号来源 | `signal_daily` 表 | `index_id='sh'` |
| 价格来源 | `index_daily` 表 | `close` 字段 |
| 数据区间 | 1991-02-05 ~ 2026-06-09 | 35.3 年 |

### 1.2 三种策略路径

| 路径 | 描述 | 买入规则 | 卖出规则 |
|------|------|----------|----------|
| 买固定1万+卖清仓 | 每次买1万，卖点清仓全部 | 1w/次，最多10笔 | 清仓全部持仓 |
| 全仓进出 | 一次一笔，买全部现金，卖清仓 | 全部现金买入 | 清仓 |
| 固定1万进出（FIFO） | 每次买1万，卖FIFO最早一笔 | 1w/次，最多10笔 | FIFO卖出1笔 |

### 1.3 三种信号组合

| 子Tab | 包含信号 | buy_types |
|--------|----------|-----------|
| 主买+卖 | buy + sell | `{"buy"}` |
| 辅买+卖 | buy_aux + sell | `{"buy_aux"}` |
| 主买+辅买+卖 | buy + buy_aux + sell | `{"buy", "buy_aux"}` |

### 1.4 连续同向信号处理

- 路径 B（全仓进出）：连续同向买入/卖出跳过
- 路径 C（买固定1w+卖清仓）：连续同向卖出跳过
- 路径 A（固定1w FIFO）：不跳过（FIFO自然处理）

### 1.7 关键代码文件

- **主脚本**：`scripts/simulate_trade.py`（~800行，三路径 + HTML生成）
- **数据源**：`data/sentiment.db`（signal_daily + index_daily）
- **统计参考**：`data/signal_stats.json`（凯利/胜率/盈亏比）
- **输出**：`static-site/trade_sim.html`

### 1.8 改造为多品种的注意事项

1. **参数化 index_id**：当前 `get_signals("sh")` 硬编码，需改为参数
2. **输出文件名**：`trade_sim_{index_id}.html`
3. **品种中文名**：标题显示中文名而非 index_id
4. **价格字段**：当前使用 `index_daily.close`，需确认所有品种在 `index_daily` 表中都有数据
5. **资金规则**：A股/港股/美股/全球指标的成交规则不同（最小交易单位、汇率等），但模拟回测可先统一用「按收盘价成交」简化
6. **特殊品种**：
   - `s.cross_market` / `s.a_sentiment`：综合情绪指标，不是可交易资产，标记为「仅参考」
   - `g.a_qvix_*`：波动率指数，非直接可交易
   - `g.cn10y` / `g.us10y`：国债收益率，非价格指数
   - `g.comex_silver` / `g.gold` / `g.wti_oil`：商品，价格单位不同

## 四、子进程执行规范

### 4.1 单品种执行步骤

1. 修改 `scripts/simulate_trade.py` 中的 `get_signals()` 调用，传入目标 `index_id`
2. 调整输出文件名 `trade_sim_{index_id}.html`
3. 运行 `python3 scripts/simulate_trade.py` 生成 HTML
4. 在 `static-site/app.js` 中添加回测入口链接（可选，后续统一处理）
5. 汇报回测结果摘要（9场景控制台输出）

### 4.2 改造方案

**方案A（推荐）**：重构 `simulate_trade.py` 为参数化脚本
- 支持 `python3 scripts/simulate_trade.py --index sh` 命令行参数
- 输出 `static-site/trade_sim_sh.html`
- 一个子进程批量处理多个品种

**方案B**：当前方式，每个品种手动改 index_id
- 简单但低效，适合少量品种

### 4.3 品种中文名映射

需要建立 index_id → 中文名 的映射表，可从以下来源获取：
- `config/indicators.yaml` 中的 `name` 字段
- `app/compute/signals.py` 中的 `INDEX_NAMES` 或类似常量
- 申万行业代码需查表（sw_801xxx → 行业名）
