# 买卖点策略深度回测报告

- 生成日期：2026-07-05
- 数据截止：2026-07-10（stock_daily.db mootdx_daily_raw + sentiment.db index_daily）
- 标的：13 主要指数 + 3 红利 + 31 申万行业 + 抽样 200 只全 A 股个股（共 244 个序列；个股从 3609 只合格标的中固定种子抽样，覆盖大小盘/行业/上市时长）
- RSI 算法：period=14, EWM α=1/14, adjust=False（复刻 app/compute/signals.py `_rsi`）
- 信号后收益：信号日 close → N 交易日后 close（无 N 日后数据则跳过）
- 买点胜率=收益>0 占比；卖点胜率=收益<0 占比（信号后下跌才算对）
- 盈亏比=平均盈利/平均亏损（绝对值）；均值收益单位 %
- 周期：全史 / 近10年(>2016-07-12) / 近3年(>2023-07-11) / 近1年(>2025-07-10)
- 持有 horizon：5 / 10 / 20 / 60 交易日
- 当前方案基线：买 C1 = RSI(14) 上穿30；卖 D1 = 20日高(high)回落5%

> **WebSearch 说明**：本次环境 WebSearch 工具多次调用均返回空结果（5 个不同查询均无内容），无法获取实时网页。下列策略清单基于经典技术分析的公认定义（教科书级、稳定），并附权威参考来源 URL（Investopedia / Wikipedia / 原始论文）。回测部分完全独立、可复现。

## 7. 落地建议（signals.py 改法）

### 7.1 候选 A（维持现状）——无需改代码

C1 买点 + D1 卖点已在 `app/compute/signals.py` 实施（2026-07-06 验收）。本回测在 244 资产上
再次验证其接近最优，**建议不动**。仅需持续监控 D1 实盘胜率（若连续 2 季度 10 日胜率<45%，
考虑启用候选 C 的 BB_upper_revert 互补）。

### 7.2 候选 B（加 BB_lower_revert 辅买点）

在 `app/compute/signals.py` `compute()` 内、现有 buy 判定后追加辅买点。
**主买点 C1 不动**，BB_lower_revert 作为低优先级辅买点单独标（避免喧宾夺主）。

BB_lower_revert 只用 close（布林带=MA20±2σ），无需改 `normalize.py`。

```python
# 在 buy = ... 之后追加（C1 主买点不变）
bu_, mid_, bl_ = bollinger(close, 20, 2.0)   # 复刻：mid=MA20, sd=std(ddof=0)
buy_aux = ((close.shift(1) < bl_.shift(1)) & (close > bl_)).fillna(False)

# reason 格式（辅买点，标 aux 区分）
for date in buy_aux[buy_aux].index:
    c = close.get(date); b = bl_.get(date); r = rsi.get(date)
    reason = f"布林下轨回归(下轨{b:.0f},close{c:.0f})"
    if pd.notna(r): reason += f',RSI={r:.0f}'
    # cross 软标签同 C1
    signals.append((date, iid, "buy_aux", reason))   # signal 字段用 buy_aux 区分优先级
```

> 注：C1 与 BB_lower_revert 信号会部分重叠（同日触发），UI 可去重或并排展示。
> 建议主买点 C1 优先级高、辅买点 buy_aux 优先级低。

### 7.3 候选 C（加 BB_upper_revert 辅卖点）

在 D1 主卖点基础上追加辅卖点（短周期止盈）。**主卖点 D1 不动**。同样只用 close，无需改 normalize。

```python
# 在 sell = ...（D1）之后追加
bu_, mid_, bl_ = bollinger(close, 20, 2.0)
sell_aux = ((close.shift(1) > bu_.shift(1)) & (close < bu_)).fillna(False)

for date in sell_aux[sell_aux].index:
    c = close.get(date); b = bu_.get(date); r = rsi.get(date)
    reason = f"布林上轨回归(上轨{b:.0f},close{c:.0f})"
    if pd.notna(r): reason += f',RSI={r:.0f}'
    signals.append((date, iid, "sell_aux", reason))
```

> D1 与 BB_upper_revert 时间维度互补（D1 强在 20d，BB_upper_revert 强在 5d/10d），
> 同日触发可视为高置信减仓信号。

### 7.4 reason 字段建议

无论哪个候选，reason 末尾保留 cross 软分级标签（`_cross_tag`），与现状一致。
辅买/辅卖点用 `buy_aux` / `sell_aux` 区分优先级，UI 可折叠展示。

## 8. 诚实声明与局限

- 信号独立统计：买/卖 forward return 各自评估，未模拟「买入→持有→卖出」真实配对收益；实际持仓收益还取决于信号匹配、滑点、手续费。
- 卖点本质难预测：A 股长期向上漂移，任何卖点都难长期高胜率；本回测在更大样本上再次印证此结构性问题。
- 个股 vs 指数：个股波动大、信号更频繁、胜率结构可能与指数不同；抽样 200 只已尽量覆盖，但仍非全市场。
- 缠论未纳入：定义主观、实现差异大，独立脚本难以可靠复刻。
- WebSearch 工具本次返回空结果，策略清单基于公认定义 + 权威 URL；如需最新研报/社区变体需后续补充检索。
