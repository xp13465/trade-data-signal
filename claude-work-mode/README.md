# Claude 通用工作模式规范包

这是一份从实际项目磨合中提取的 **通用工作模式规范**，描述「怎么和 Claude Code 高效配合」的元规范。与具体项目业务无关，可移植到任何新项目或给其他人复用。

## 包内文件

- `CLAUDE.md` — 规范正文（11 条 + 工作流总则），复制到 `~/.claude/CLAUDE.md` 即生效。
- `README.md` — 本说明文件。

## 怎么用

### 给自己新项目用（全局，所有项目自动加载）

```bash
cp CLAUDE.md ~/.claude/CLAUDE.md
```

Claude Code 启动时会自动加载 `~/.claude/CLAUDE.md`，对所有项目生效。无需逐项目配置。

### 给单个项目用（仅该项目生效）

```bash
cp CLAUDE.md /path/to/your-project/CLAUDE.md
```

放在项目根目录，Claude Code 在该项目内会自动读取。

### 给其他人用

让对方把 `CLAUDE.md` 内容贴到他自己的 `~/.claude/CLAUDE.md`（全局）或项目根 `CLAUDE.md`（单项目）即可。

## 包含的规范清单

每条一句话：

1. **监管 + loop 工作模式** — 主上下文派子 agent 干活 + 独立验收，只读汇报不自己上手，子 agent 每个任务新开不复用。
2. **不问 yes/no，自行验收连轴转** — 能自决的自决，只在真·方向性分叉时给选项。
3. **不烧 token** — 直接给判断和动作，不写自问自答的长串权衡。
4. **始终用中文回复** — 无论 compact 与否、摘要什么语言，回复一律中文。
5. **调研后给方案，方向分叉才问** — 技术细节自己调研定，多路径需拍板才列选项。
6. **参数优化测试驱动** — 测多方案 + 数据报告让用户选，不凭空问参数。
7. **候选不硬选** — 匹配到多个全列出排序让用户选，匹配不到留空不硬塞代理。
8. **新功能先隔离，验证后再融合** — 功能 tab + 代码数据物理隔离，不污染线上。
9. **改完必须 git push** — 改动完成立即 commit + push，漏推 = 白干。
10. **遇 API 错误不卡死** — 换通道或暂存任务，能力与模式匹配，别硬试不支持的调用。
11. **改静态资源必须破缓存** — 改 CSS/JS 后更新版本号，否则浏览器/CDN 缓存旧版。

## 重要说明

- **这是通用工作模式**，不含任何项目特定知识（业务逻辑、数据口径、采集脚本、策略细节等）。
- **项目特定知识** 靠各项目自己的 `REQUIREMENTS.md` / `NOTES.md` / `TASKS.md` 沉淀，开工先读这三份。
- 本规范是「磨合出来的协作约定」，不是 Claude Code 的官方配置；可根据团队实际增删调整。
- 复制到全局 `~/.claude/CLAUDE.md` 后，若个别项目有特殊要求，可在项目根 `CLAUDE.md` 里追加覆盖。
