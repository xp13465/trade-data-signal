## BUG-001：手动补录日期默认值格式错误

| 属性 | 值 |
|------|-----|
| **优先级** | 🔴 高 |
| **模块** | 前端 / app.js |
| **文件** | `static/app.js` |
| **行号** | 第 243 行 |
| **状态** | 🔴 待修复 |

### 问题描述

打开"手动补录"弹窗时，日期输入框（`<input type="date">`）的值始终为空。原因是代码将当天日期格式化为 `yyyyMMdd`（无横杠）后赋值给 date input，但 HTML5 date input 要求格式必须是 `yyyy-MM-dd`（带横杠）。

### 根因代码

```javascript
// static/app.js 第 243 行
document.getElementById("m-date").value = new Date().toISOString().slice(0, 10).replace(/-/g, "");
//                                                                                ^^^^^^^^^^^^^^^^^^^^^^^^
//                                                                                BUG: 去掉了横杠
```

`new Date().toISOString().slice(0, 10)` 产生 `"2026-07-06"`，经过 `.replace(/-/g, "")` 变成 `"20260706"`，HTML5 date input 无法识别此格式，赋值失败。

> 注意：第 248 行提交时 `.replace(/-/g, "")` 是**正确的**（后端 API 期望 `yyyyMMdd` 格式），不需要改。

### 复现步骤

1. 打开看板页面
2. 点击"＋ 手动补录"按钮
3. 观察日期输入框 → 为空
4. 浏览器 Console 出现 WARNING: `The specified value "20260706" does not conform to the required format, "yyyy-MM-dd"`

### 修复方案

```javascript
// 修改第 243 行，去掉 .replace(/-/g, "")
document.getElementById("m-date").value = new Date().toISOString().slice(0, 10);
```

### 验证方法

1. 修复后打开手动补录弹窗
2. 确认日期输入框自动填充为当天日期（格式 yyyy-MM-dd）
3. 确认 Console 无 WARNING
4. 提交后确认后端收到的是 yyyyMMdd 格式（第 248 行的 replace 保证）

## BUG-004：手动补录接口无日期格式校验

| 属性 | 值 |
|------|-----|
| **优先级** | 🔴 高 |
| **模块** | 后端 API |
| **接口** | `POST /api/manual` |
| **状态** | 🔴 待修复 |

### 问题描述

提交手动补录时，`date` 字段未校验格式。传入非 `yyyyMMdd` 格式的字符串（如 `"invalid"`）也能提交成功，返回 `{"ok":true}`，可能破坏数据一致性。

### 复现步骤

```bash
curl -s -X POST "http://e833cb64.natappfree.cc/api/manual" \
  -H "Content-Type: application/json" \
  -d '{"date":"invalid","metric_id":"a_width_up_count","value":100}'
# 返回: {"ok":true}  ← 应该返回 400 错误
```

### 修复建议

```python
import re
from datetime import datetime

DATE_PATTERN = re.compile(r'^\d{8}$')

@app.post("/api/manual")
async def manual_entry(body: ManualEntry):
    if not DATE_PATTERN.match(body.date):
        raise HTTPException(status_code=400, detail="日期格式错误，要求 yyyyMMdd（如 20260703）")
    try:
        datetime.strptime(body.date, "%Y%m%d")
    except ValueError:
        raise HTTPException(status_code=400, detail="无效的日期")
    # ... 继续处理
```

### 验证方法

```bash
# 修复后应返回 400
curl -s -X POST "http://e833cb64.natappfree.cc/api/manual" \
  -H "Content-Type: application/json" \
  -d '{"date":"invalid","metric_id":"a_width_up_count","value":100}'
# 期望: {"detail":"日期格式错误，要求 yyyyMMdd"} + HTTP 400
```

```javascript

  if (!body.date || !body.metric_id || isNaN(body.value)) return alert("请填完日期/指标/数值");
  
  // 检查是否已有数据
  const existing = await fetchJSON(`/api/manual/check?date=${body.date}&metric_id=${body.metric_id}`);
  if (existing.exists && !confirm(`日期 ${body.date} 的 ${body.metric_id} 已有数据（${existing.value}），是否覆盖？`)) {
    return;
  }
  
  // ... 继续提交
};
```
