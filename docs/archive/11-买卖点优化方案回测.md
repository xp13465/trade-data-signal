# 买卖点优化方案回测报告（卖点降噪 + 买点增强）

- 生成日期：2026-07-05
- 数据截止：2026-07-07（sentiment.db index_daily + stock_daily.db mootdx_daily_raw）
- 标的：13 主要指数 + 31 申万行业 + 抽样 200 只个股（共 244 资产；个股从合格标的中固定种子 20260705 抽样，与 08 报告同款样本）
- RSI 算法：period=14, EWM α=1/14, adjust=False（复刻 app/compute/signals.py `_rsi`）
- D1：close 从近 N 日 high 之 max 回落 P%（high-based，事件化 shift(1)）
- BB：MA20 ± 2σ（std ddof=0）
- 信号后收益：信号日 close → N 交易日后 close（无 N 日后数据跳过）
- 买点胜率=收益>0 占比；卖点胜率=收益<0 占比（信号后下跌才算对）
- 盈亏比=平均盈利/平均亏损（绝对值）；均值收益单位 %
- 周期：全史 / 近3年(>2023-07-08) / 近1年(>2025-07-07)；horizon: 5/10/20 日
- 信号密度=总信号 / (资产数 × 平均年限) = 信号/资产/年（越低噪声越少）；本次 n_assets=244，平均年限 17.8 年
- 现状基线：买 C1_30 = RSI(14) 上穿30；卖 D1_20_5 = 20日高回落5%

> 用户反馈：卖点很多买点少，卖点噪点多。本报告回测卖点降噪（S1-S5）+ 买点增强（B1-B2）+ 组合方案，给数据依据与推荐。所有指标自复刻，不 import app。

## 7. 落地建议（signals.py 改法）

### 7.1 推荐方案：B1_dual + S1_MA60bull

在 `app/compute/signals.py` `compute()` 内实施。**主买点 C1 + 主卖点 D1 触发逻辑可保留**，降噪/增强通过追加过滤或辅信号实现。

```python
# === 买点增强（B1_dual）：C1 不动，追加 BB下轨回归辅买点 ===
bu_, mid_, bl_ = bollinger(close, 20, 2.0)   # mid=MA20, sd=std(ddof=0)
buy_aux = ((close.shift(1) < bl_.shift(1)) & (close > bl_)).fillna(False)
# reason 标 'buy_aux' 区分优先级；C1 主买点 signal='buy' 不变

# === 卖点降噪 ===
# S1_MA60bull：D1 触发后，仅当 close>MA60 才发出卖信号
ma60 = close.rolling(60, min_periods=60).mean()
sell = sell & (close > ma60)   # sell=D1 原掩码，叠加趋势过滤
```

> 注：若推荐组合含 B1_dual，辅买点用 `buy_aux` 区分优先级，UI 可折叠展示；C1 与 BB下轨回归同日触发时去重或并排展示。reason 末尾保留 cross 软分级标签。

### 7.2 备选方案（若用户偏好不同）

- **纯降噪（不动买点）**：C1_30 + S1_MA60bull 或 C1_30 + S2_profit_C1。改动最小，只过滤卖点，买点不变。适合「只想减少卖点噪声」的场景。
- **纯增强（不动卖点）**：B1_dual + D1_20_5。加辅买点，卖点不变。适合「想捕捉更多反弹」。
- **最激进降噪**：B1_dual + S5_D1andBB。双重确认卖点+双买点，信号最少但最精准；风险是样本量下降、可能漏掉有效信号。

## 8. 诚实声明与局限

- 信号独立统计：买/卖 forward return 各自评估，未模拟「买入→持有→卖出」真实配对收益（见 10 报告：配对回合均值+0.67%/胜率44.6%/年化+2.52%）。
- 卖点本质难预测：A 股长期向上漂移，任何卖点近年盈亏比<1（结构性问题），降噪能减噪但不能让卖点变「好」。
- S2 盈利约束依赖前买点游标，若长期无买点则卖点被全滤（强势市可能0卖点）。
- 信号密度受资产年限影响（个股年限<指数），密度值仅作横向对比。
- 抽样 200 只个股已覆盖大小盘/行业，但仍非全市场；缠论等主观策略未纳入。
