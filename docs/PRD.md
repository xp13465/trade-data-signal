## 1 产品定位

### 1.1 是什么

**A股情绪看板（trade-data-signal，简称 tdsignal）** 是一个面向 A 股个人投资者的盘后复盘与情绪量化工具。它聚合多源市场数据（新浪/腾讯/baostock/mootdx/东财/申万），通过情绪评分模型、买卖点信号策略、行业宽度热力图、期货机构持仓等模块，将分散的市场数据转化为可解读的情绪温度计和趋势信号。

### 1.2 给谁用

- A 股散户/个人投资者：盘后快速复盘当日市场情绪、行业轮动、买卖点触发情况
- 量化爱好者：验证经典策略（RSI/布林/Donchian/MACD/Supertrend 等）在 A 股的历史回测表现
- 趋势跟踪/均值回归交易者：参考冰点/过热情绪分、买卖点信号做择时辅助

### 1.3 解决什么问题

| 痛点 | 解决方案 |
|------|----------|
| 市场情绪难以量化（涨停家数/炸板/成交量各自孤岛） | 6 分量加权综合情绪分（0-100）+ 8 分量恐贪指数 |
| 买卖点信号散落在各软件、无统一回测验证 | C1(RSI30) + B1(BB下轨回归) + D1(20日高回落5%) 统一策略，244 资产全史回测 |
| 行业轮动节奏不可视 | 申万 31 行业 mini 折线 + 涨跌幅热力图 + 轮动速度卡片 |
| 散户看不懂专业指标 | 散户化文案（"🧊 冰点"/"🔥 过热"）+ 凯利仓位折叠详解 + 一句话总结横幅 |
| 机构动向不透明 | 中金所前 20 会员净多空持仓 + 同向/逆向准确率统计 |

### 1.4 不做什么

- 不做实时行情（数据盘后 T+0/T+1 采集，非实时推送）
- 不做个股推荐/投资建议（信号为统计参考，不构成交易指令）
- 不做自动化交易（仅展示+回测，无下单接口）
- 不做社区/UGC（纯数据工具，无用户生成内容）

## 2 功能架构

### 2.1 整体架构

站点为单页应用（SPA），顶部 5 个主 Tab 切换：

| Tab | 名称 | H5 图标 | 核心功能 |
|-----|------|---------|----------|
| overview | 📊 概览 | 📊 | 一句话总结 + 恐贪指数 + 综合情绪分 + 冰点/买卖点卡片 + 行业热力图 + 轮动速度 |
| market | 📈 大盘 | 📈 | A 股/港股/全球指数，含涨停/资金流/宽度/QVIX/龙虎榜/解禁等 |
| sentiment | 😊 综合情绪 | 😊 | 冰点过热热力图 + 恐贪指数 + 6 指数情绪分 + 跨市场评分 + 期货持仓 |
| industry | 🏭 行业概念 | 🏭 | 申万 31 行业折线网格 + 概念板块 + 热力图 + 轮动速度 |
| lab | 🧪 策略实验室 | 🧪 | 22 策略回测矩阵 + 模拟配对回测 + 推荐榜 + 买卖信号弹窗 |

### 2.2 双轨部署

| 轨道 | 目录 | 服务 | 用途 |
|------|------|------|------|
| 动态版 | `web/` | Mac 本地 FastAPI（:8000） | 开发调试 + 动态 /api/ 接口 |
| 静态版 | `static-site/` | Cloudflare Pages | 公网生产，./data/ 静态 JSON |

两版 JS/CSS 同源同构，仅数据路径不同（动态 /api/ vs 静态 ./data/）。每次采集后经 `deploy.sh` 同步导出 JSON 到双版。

### 2.3 数据流

```
数据源(baostock/mootdx/akshare/申万/腾讯)
  → collector/runner.py(11步采集)
  → SQLite(data/sentiment.db, WAL模式, 11张表)
  → compute/runner.py(15步计算)
  → JSON导出(web/data/ + static-site/data/)
  → 前端渲染(ECharts + 原生JS)
  → git commit + push → Cloudflare Pages 自动部署
```

### 2.4 采集调度

| 时间 | 任务 | 说明 |
|------|------|------|
| 15:33 | `update_all.sh` 全量采集 | 4 条并行 pipeline(core/width/futures/stock_daily)，各自采集→计算→导出→push |
| 20:00 | `backfill_indices.sh` 补采兜底 | 校验 9 核心指数 + 31 申万行业指数缺失，多源补采后重算情绪分 + push |

非交易日跳过采集，仅 deploy + check_signals。

## 3 详细功能模块

### 3.1 概览 Tab（overview）

**一句话总结横幅**：盘后复盘入口，自动生成当日市场摘要文案 + 情绪标签 + 恐贪标签 + 冰点标记 + 日期 + 采集时间。支持展开历史收盘分析弹窗（分页浏览往日摘要）。

**核心指标 KPI 卡片行**：
- 恐贪指数（0-100）+ 标签（极度恐惧/恐惧/中性/贪婪/极度贪婪）
- A 股综合情绪分（0-100）+ 标签（极度悲观→亢奋 7 档）
- 跨市场综合评分（0-100）
- 涨停/跌停家数
- 成交额（亿）
- 成交量比（vs 20 日均量）

**冰点日卡片**：按日分组展示近期情绪冰点（≤20）的指数，同日 4 个/行，今日高亮。可点击弹窗查看走势图 + 信号。

**买卖点卡片**：按日分组展示近期 buy/buy_aux/sell 信号，今日高亮。买卖点可点击弹窗展示走势图 + 信号 markPoint + 时间段切换（全/10y/5y/3y/1y）。

**指数 mini 折线网格**：8 个 A 股宽基指数（sh/sz/hs300/sz50/csi500/csi1000/cyb/kc50）的 sparkline + 涨跌幅 badge + 买卖点 markPoint。

**行业涨跌幅热力图**：申万 31 行业近 1 日 / 近 5 日 / 全部切换，红涨绿跌（A 股惯例）。

**板块轮动速度卡片**：申万行业 + 概念板块的 5d/10d/20d 轮动速度（0-100%），快速/中等/缓慢三档 + 大白话解释折叠。

**均线排列卡片**：MA5/10/20/60 多头排列/空头排列/交叉统计 + 8 指数明细表。

**市场位置感卡片**：8 指数近 1y/3y/5y 价格分位（低位/偏低/合理/偏高/高位 5 档）+ 进度条。

**新高新低卡片（NH-NL）**：8 指数创 52 周新高/新低数量 + NH-NL 差值。

**AD Line（腾落线）**：累积涨跌家数 + 涨跌比 + MA5/MA20。

**成交量比信号**：放量上涨/放量下跌/缩量上涨/缩量下跌/正常 5 档分类。

### 3.2 大盘 Tab（market）

二级 Tab 切换：A 股 / 港股 / 全球。

**A 股子页**：
- 涨停/跌停/炸板/封板率统计
- 连板梯队（连板数 × 个股数）
- 市场宽度（上涨/下跌/涨停/跌停/炸板 历史折线）
- 主力资金流（净流入/流出趋势）
- 成交额 + 成交量比
- QVIX（中国版波动率指数，a_qvix_300 / a_qvix_1000）
- 股息率
- 龙虎榜
- 解禁/IPO 日历
- 港股通净买入

**港股子页**：恒生指数/恒生科技/恒生国企等 mini 折线 + 涨跌幅。

**全球子页**：标普500/纳指/道指/日经/德国DAX/英国富时等全球指数 mini 折线 + 美债收益率/原油/黄金/美元兑离岸人民币等宏观指标。

### 3.3 综合情绪 Tab（sentiment）

**冰点/过热热力图**：X 轴日期 × Y 轴 6 指数（上证50/沪深300/中证500/中证1000/创业板/科创50），色块红(冰点≤20)/绿(过热>80)/灰(中性)。标题显示最新日期冰点/过热计数。

**恐贪指数**：8 分量等权（a_sentiment + cross_market + 6 per-index 情绪分），0-100，visualMap 5 档色（极度恐惧红→极度贪婪绿）。

**A 股综合情绪分**：6 分量加权（涨跌比 25% + 涨停 20% + 炸板 15% + 连板 15% + 成交额 10% + 北向 15%），0-100。

**6 指数 per-index 情绪分**（按散户关注度排序：中证1000→创业板→科创50→中证500→沪深300→上证50）：
- 每个指数独立情绪分（RSI + 量能偏离 + 涨跌幅 + QVIX 等权）
- 0-100，≤20 冰点红 / >80 过热绿 / 20-80 中性蓝

**跨市场综合评分**：type=simple + enabled 的指标，trim_mean（去掉最高最低后取均值，≥3 个指标），0-100。

**期货机构持仓**（通过 /api/futures 异步加载）：
- 昨日净多空概览表（机构前20 / 中信期货 / 国泰君安 × IF/IC/IH/IM/综合）
- 历史同向/逆向准确率表（30d/60d/120d 滚动窗口）
- 综合净多空手数折线图（3 条线 + hover 显示准确率）
- 每角色各品种净多空折线图（4 张）

### 3.4 行业概念 Tab（industry）

**板块轮动速度卡片**（最先展示，判断行情性质）。

**锚点导航条**：申万行业 / 概念板块 快速跳转 + 回到顶部。

**申万一级行业涨跌幅热力图**：近 1 日 / 近 5 日 / 全部切换，红涨绿跌，按涨幅排序，格内显示百分比数值。

**行业搜索条**：输入名称关键词实时过滤行业网格。

**申万行业指数折线网格**（3 列 PC / 1 列 H5）：
- 每个行业卡片：名称 + ETF 标签（点击复制代码，悬浮弹全部候选按成交额排序）+ 最新收盘价 + 涨跌幅 badge
- mini 折线图（含买卖点 markPoint）
- 回测统计 hint（胜率/盈亏比/均值/样本数 + 凯利仓位 + 频率 hover popup）
- 资金流 / 成交额 / 换手率 mini sparkline（3 条）
- 行业内宽度 mini chart（涨跌家数堆叠 + 涨停/跌停/炸板 hover）

**概念板块指数折线**（同上结构，含买卖点 + 回测统计）。

## 4 用户场景

### 4.1 盘后复盘（核心场景）

1. 收盘后打开公网看板，概览 Tab 一句话总结横幅快速了解当日市场全貌
2. 查看恐贪指数/情绪分判断市场温度（冰点=潜在反弹机会，过热=警惕回调）
3. 冰点日卡片查看哪些指数进入冰点区域（≤20）
4. 买卖点卡片查看今日触发的 buy/buy_aux/sell 信号，点击弹窗看走势图
5. 行业热力图查看今日领涨/领跌行业
6. 板块轮动速度判断行情性质（快速轮动=无主线谨慎追高，缓慢=主线明确可跟）

### 4.2 选股参考

1. 行业概念 Tab 查看申万 31 行业折线 + 资金流/换手率/宽度
2. 点击 ETF 标签复制相关 ETF 代码（按成交额排序多候选）
3. 行业内宽度 mini chart 判断行业内个股涨跌结构
4. 买卖点 markPoint 在行业折线上标注历史信号点位

### 4.3 策略验证

1. 策略实验室 Tab 查看 22 策略回测矩阵
2. 点击策略卡片进入详情，查看指标图表 + 信号点位
3. 模拟配对回测：选指数 → 选时间窗口 → 切换全仓/定额模式 → 查看净值曲线 + 交易记录
4. 推荐榜按 score 排序查看最优配对
5. 买卖信号弹窗对比买/卖策略在同时期的信号点位

### 4.4 机构动向追踪

1. 综合情绪 Tab 底部期货机构持仓
2. 昨日净多空概览表查看机构/中信/国君在 IF/IC/IH/IM 的净持仓方向
3. 历史准确率表判断跟随机构方向的胜率（同向 vs 逆向）
4. 折线图 hover 查看每日净持仓 + 当日准确率

## 5 数据源

### 5.1 数据源一览

| 数据源 | 库/接口 | 采集内容 | 特点 |
|--------|---------|----------|------|
| baostock | `baostock` Python 包 | A 股指数日线（OHLC+pct+amount） | T+1 延迟，7/8 核心指数（缺科创50），串行 |
| mootdx | `mootdx` Python 包 | 全 A 股日线（5072 只） | TCP 7709 不封 IP，D1 主力源，通达信协议 |
| 新浪（akshare） | `akshare.stock_zh_index_daily` | A 股指数日线 + 全市场宽度快照 | 全覆盖，主源，快 |
| 腾讯（akshare） | `akshare.stock_zh_index_daily_tx` | A 股指数日线（全量历史） | 全覆盖但慢（~12s/只），无 pct 需自算，兜底 |
| 东财（akshare） | `akshare.index_zh_a_hist` 等 | 行业资金流/换手率/龙虎榜 | **IP 封禁风险**（clist 端点被封），fflow/kline 端点可用 |
| 申万官方 | `swsresearch.com` trend API | 申万 31 行业指数 OHLC | 唯一支持 sw_* 代码的源，T+1 发布延迟 |
| 同花顺 | akshare 概念板块接口 | 概念板块涨跌/资金流 | 27 个概念板块 |

### 5.2 各源采集职责

- **新浪主源**：指数日线（step2 indices）+ 单值指标快照（step1 metrics）+ 板块涨跌（step3 boards）
- **baostock 备用**：指数补采（7/8 核心，缺科创50）+ 全 A 股日线备用（默认跳过，需 `RUN_BAOSTOCK=1`）
- **腾讯兜底**：指数补采（全覆盖，慢，仅 baostock 补不到时用）
- **mootdx D1 主力**：全 A 股日线增量更新（step7 mootdx），TCP 7709 不封 IP
- **东财**：行业资金流 + 换手率（step4 industry_extras，用 fflow/kline 端点非 clist 不被封）
- **申万官方**：申万 31 行业指数 OHLC（step2 indices 的 sw_* 部分，T+1 延迟）
- **akshare 聚合**：港股/全球指数/QVIX/美债/原油/黄金/汇率等宏观指标

### 5.3 多源补采策略

指数采集后自动校验 9 核心 A 股指数 + 31 申万行业指数是否到位：

```
新浪主源未取到 → baostock（7/8，缺科创50）→ 腾讯兜底（全覆盖，慢）
申万 sw_* → 申万官方 trend API（唯一源，T+1 延迟则跳过等下次补）
```

## 6 部署架构

### 6.1 本地开发环境

- **OS**：macOS（Darwin 23.4.0）
- **Python**：3.11+（.venv 虚拟环境）
- **框架**：FastAPI + Uvicorn

- **前端**：原生 JS + ECharts（无框架），CSS + HTML
- **调度**：macOS launchd（com.trade.update-all + com.trade.backfill-evening）

### 6.2 公网部署

- **托管**：Cloudflare Pages
- **静态文件**：static-site/ 目录 → Cloudflare Pages 自动部署（git push 触发）
- **数据**：static-site/data/*.json（采集后导出 + commit + push）
- **域名**：tdsignal-ujpzw01zm.maozi.io

### 6.3 采集流水线

`update_all.sh` 并发启动 4 条 pipeline（各自独立采集→计算→导出→commit+push）：

| Pipeline | 内容 | 阻塞性 | 说明 |
|----------|------|--------|------|
| core | 指数/指标/情绪分/信号 | 分钟级先上线 | 快核心，不依赖 mootdx |
| width | mootdx/行业宽度/全市场宽度 | 慢（5072只） | 完成后覆盖上线 |
| futures | 期货机构持仓 + 准确率 | 独立 | 独立上线 |
| stock_daily | 全 A 股日线备用源 | 后台 | 不 export 不 push，不阻塞 |

进程互斥：`fcntl.flock --nb` 非阻塞锁，重复跑自动跳过。

### 6.4 数据库结构

SQLite `data/sentiment.db`，11 张表：

| 表名 | 用途 |
|------|------|
| daily_metric | 每日指标值（metric_id, date, value, source） |
| index_daily | 指数日线（index_id, date, OHLC, pct_change, amount） |
| board_daily | 板块日线（board_type, board_name, pct_change, net_inflow） |
| score_daily | 情绪评分（score_id, date, value） |
| signal_daily | 买卖点信号（index_id, date, signal, reason） |
| manual_entry | 手动补录（source='manual' 不被自动覆盖） |
| collect_log | 采集日志（date, metric_id, status, message） |
| alert_log | 告警日志 |
| industry_width_daily | 行业内宽度（涨跌/涨停/炸板） |
| futures_position | 期货持仓（date, role, product, net_position） |
| futures_accuracy | 期货准确率（date, role, window, follow, contrarian） |
